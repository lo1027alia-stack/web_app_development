# 食譜收藏夾系統 — 產品需求文件（PRD）

> **版本**：v1.0　｜　**建立日期**：2026-04-15　｜　**狀態**：草稿審閱中

## 1. 專案概述

### 1.1 背景與動機

現代家庭工作者常常需要花費大量時間在網路上搜尋食譜，卻缺乏一個統一且個人化的地方來整理與收藏喜愛的料理。食材買了卻忘了步驟、影片教學找不到、常煮的菜和想嘗試的菜混在一起——這些都是日常煮飯的痛點。

「食譜收藏夾」旨在提供一個簡單、直覺的個人食譜管理平台，讓使用者能輕鬆新增、分類、收藏食譜，並隨時取用。

### 1.2 目標用戶

| 用戶類型 | 描述 |
|---|---|
| **主要用戶** | 家庭工作者（每日負責備餐的家庭成員） |
| **次要用戶** | 對料理有興趣、想自我紀錄的個人使用者 |

### 1.3 核心價值主張

- ✅ **一站式收藏**：不再分散在書籤、筆記、截圖中，統一管理個人食譜庫
- ✅ **快速找菜**：依菜系分類快速定位，加上喜愛標記讓常用食譜一目了然
- ✅ **完整紀錄**：食材清單、步驟說明、影片連結全都在同一個頁面

## 2. 功能需求

### 2.1 主要功能列表

#### 功能一：菜系分類管理

> **使用者故事**：作為家庭工作者，我希望能依菜系（如：台式、日式、義式、中式、韓式）來分類食譜，以便快速找到特定風格的料理，不需要一筆一筆翻找。

- 支援自訂菜系標籤（新增、編輯、刪除）
- 每道食譜可歸屬於一個或多個菜系
- 首頁可依菜系篩選食譜列表

---

#### 功能二：食材清單與烹飪步驟

> **使用者故事**：作為家庭工作者，我希望在每道食譜頁面看到完整的食材清單與步驟說明，以便在備餐時直接按步驟操作，不需要額外查閱其他資料。

- 食材以條列方式填寫（名稱、份量、單位）
- 烹飪步驟依序編號，支援多步驟輸入
- 顯示時有清晰的排版與視覺分隔

---

#### 功能三：新增與編輯食譜

> **使用者故事**：作為家庭工作者，我希望能快速新增自己找到的食譜，並在食材或步驟有誤時能隨時編輯，以便保持食譜資料的準確性。

- 提供新增食譜的表單頁面（標題、菜系、食材、步驟、影片連結）
- 支援編輯已存在的食譜
- 支援刪除食譜（含確認提示）

---

#### 功能四：普通食譜 vs. 喜愛食譜

> **使用者故事**：作為家庭工作者，我希望能標記常煮或特別喜歡的食譜為「喜愛」，並在首頁快速切換「全部食譜」與「我的最愛」，以便優先看到最常用的料理。

- 每道食譜可切換「普通」／「喜愛」狀態（愛心圖示 toggle）
- 首頁提供「全部」、「喜愛」分頁或篩選功能
- 喜愛食譜在列表中有視覺區別（如星星、愛心標記）

---

#### 功能五：影片教學連結

> **使用者故事**：作為家庭工作者，我希望能替食譜附上 YouTube 或其他影片教學的連結，以便在烹飪時直接點開觀看，不需要再去搜尋。

- 食譜表單中提供影片 URL 欄位（選填）
- 食譜詳情頁顯示可點擊的影片連結
- 若為 YouTube 連結，嘗試嵌入預覽縮圖（Nice to Have）

---

### 2.2 輔助功能

| 功能 | 說明 |
|---|---|
| 食譜搜尋 | 依名稱關鍵字搜尋食譜 |
| 封面圖片上傳 | 為食譜新增一張封面照片 |
| 烹飪時間標記 | 標記預估料理所需時間（分鐘） |
| 份量設定 | 標記幾人份的食材份量 |

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 選型 |
|---|---|
| 後端框架 | Python Flask |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite（單機使用無需複雜部署） |
| 前端 | HTML + Vanilla CSS + 少量 JavaScript |
| 部署環境 | 本機執行（localhost），未來可擴充至雲端 |

### 3.2 效能考量

- 頁面載入時間應在 **2 秒內**完成（本機環境）
- 食譜列表支援分頁（每頁最多 20 筆），避免一次載入過多資料
- 圖片若有上傳功能，需限制單檔大小（建議 ≤ 5MB）

### 3.3 安全考量

- 防止 SQL Injection（使用 ORM 或 Parameterized Query）
- 上傳檔案需驗證副檔名（限圖片格式）
- 表單需有基本的必填驗證（前端 + 後端雙重）

### 3.4 可用性

- 介面使用繁體中文
- RWD 響應式設計，手機與平板均可流暢操作
- 關鍵操作（刪除）需有確認提示，避免誤操作
